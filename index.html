Паттерны

Паттерн - способ решения часто встречаемых задач

  1. Pub/Sub
  2. EventEmitter

Pub/Sub

Плюсы

  1. Простой
  2. Помогает разделять код

Минусы

  1. Может вырости до слишком больших размеров
  2. Возможны коллизии в событиях

EventEmitter

Плюсы

  1. Более селективные события
  2. Помагает архитектурно структурировать код
  3. Очень популярен

Минусы

  1. Предпологает использование наследования
  2. Возможна гонка событий

DOM

События в браузере реализованны через EventEmitter

  3. Всплытие

Что мы уже знаем?

  1. Javascript выполняется в одном потоке
  2. Асинхронные вызовы реализованны с помощью Event Loop
  3. Используем паттерны Pub/Sub и EventEmitter для
    1. Для декомпозиции разных частей программы
    2. Как способ синхронизации событий
